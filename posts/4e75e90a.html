简历制作

一、主体框架

  1. 基本信息
  2. 教育经历
  3. 个人经历
  4. 个人优势
  5. 获奖经历

二、重要填写部分

个人优势：

主要体现你掌握的技能，除了Web方向，尽量补充一些流量分析、安全设备、内网渗透（提权、权限维持、横向）的内容，以及一些漏洞绕过方式。

项目经验：

项目经验可以写工作里接触到的项目（渗透测试、红蓝攻防（护网）、应急响应、溯源、重保、等保等等）。

如果没有项目经验的同学主要写漏洞挖掘和漏洞复现。

漏洞挖掘可以写：公益SRC、企业SRC、众测项目等。

漏洞挖掘的案例不要写太细节，主要把成果展示出来即可。

例如：xx SRC项目

1、xxx网站xxx页面（功能点）存在xxx漏洞，评级高危，奖金xxx(公益src不用写奖金，奖金几百以上的可以写出来)

2、xxx网站xxx页面（功能点）存在xxx漏洞 评级xxxx

例举5个左右比较有代表性的成果即可，如果漏洞数量比较大，可以汇总为共挖掘漏洞xxx个，其中高危xxx个，中危xxx个。

漏洞复现：入门靶场（sqli-labs、dvwa、pikachu）这类的就不要写上去了，主要写一些CVE的漏洞复现，vulhub靶场复现，理解原理和思路。

参考《CVE漏洞复现》课程、《vulnhub靶场实战》课程、《Hack The Box》课程。

三、注意事项

  1. 选用的模板清晰明了，不要花里胡哨，以简洁风格为主（指的是模板）
  2. 照片一定有，尽量用证件照（P一下，蓝底），不要用生活照特别是自己自拍的照片。
  3. 主要去写工作经验/项目经历，在xxx的项目中，我担任什么角色，通过什么技术，最终得到什么成果，获得什么成就（可以写漏洞挖掘、攻防演练、重保值守、CTF赛事等等（主要是安全方面））
  4. 尽量多用安全方面的专有词汇去描述。
  5. 可以写一些最近火热的漏洞（Spring，log4j2）为项目经历，关注威胁情报

四、总结

  • 对于没有工作经验的，主要把项目经验写好
  • 一定要多挖漏洞，大型SRC的挖不到，公益SRC也要去挖。
  • 一定要学习了解内网渗透、应急响应、溯源等内容，并在简历上体现出来
  • 如果能拿到漏洞证书比如CNVD、EDUSRC之类的证书也是加分项

简历是找工作的第一步，也是敲门砖，俗话说人靠衣装，马靠鞍。

简历就是你的衣装，简历一定要体现你的闪光点。

十大漏洞

sql

原理：没有对用户输入项进行验证和处理直接拼接到查询语句中

利用：查询语句中插⼊恶意SQL代码传递后台sql服务器分析执行

分类：

联合注⼊、报错注⼊、布尔盲注、时间盲注、堆叠注入、⼆次注入

报错注入函数：extractvalue()、floor()、updatexml()

时间盲注函数：sleep()、BENCHMARK(count, expression)

危害：

数据库信息泄露恶意操作、网页篡改挂马、服务器被植入后门

如何防范：关闭错误提示、加waf、对输入进行过滤、限制输入长度、限制好数据库权限，drop/create/truncate等权限谨慎grant

csrf

原理：跨站点请求伪造。盗用用户身份，以用户的名义发送恶意请求

危害：

  1. 对网站管理员进行攻击
  2. 修改用户网站上的用户数据
  3. 账户劫持
  4. 传播csrf蠕虫进行大规模攻击
  5. 利用csrf进行拖库（从数据库导出数据）

如何防护：

  1. 尽量使用post，限制get
  2. 浏览器cookie策略
  3. 加验证码

文件包含

类型：

  1. 本地文件包含
  2. 远程文件包含：即加载远程文件、在php.ini中开启allow_url_include、allow_url_fopen选项。开启后可以执行任意代码

php文件包含函数：

  1. include()：代码执行到此函数时才将文件包含进来，发⽣错误时只警告并继续执行
  2. include_once()：功能和前者一样，区别在于调用同一文件时，程序只调用一次
  3. require()：立即调用此函数包含文件，发⽣错误时，会输出错误信息 并⽴即终⽌程序
  4. require_once()：功能和前者一样，区别在于调用同一文件时，程序只调用一次

利用：

  1. 读取敏感文件
  2. 远程包含shell
  3. 图⽚上传并包含图⽚shell
  4. 使⽤伪协议
  5. 包含⽇志⽂件GetShell
  6. 截断包含

修复方案：

  1. 禁⽌远程⽂件包含allow_url_include=off
  2. 配置open_basedir=指定⽬录，限制访问区域
  3. 过滤../等特殊符号
  4. 修改Apache⽇志⽂件的存放地址
  5. 开启魔术引号magic_quotes_qpc=on
  6. 尽量不要使⽤动态变量调⽤⽂件，直接写要包含的⽂件

文件上传

原理：对上传文件无限制后缀以及类型或者处理缺陷、导致越过本身权限想服务器上马

危害：操作木马文件提权、获取网站权限

绕过方法：

  1. 黑名单
    1. 后缀名不完整 .php5 .phtml等
    2. 上传.htacess
    3. ⼤⼩写
    4. 在数据包中 后⽂件缀名前加空格
    5. 后缀名前加.
    6. 加上::$DATA
    7. 未循环验证，可以使⽤x.php..类似的⽅法
  2. 白名单（一般需配合其他漏洞一起利用）
    1. %00截断
    2. 图⽚⻢
    3. 条件竞争

修复方案：

  1. 后端验证：采⽤服务端验证模式
  2. 后缀验证：基于⽩名单，⿊名单过滤
  3. MIME验证：基于上传⾃带类型艰检测
  4. 内容检测：⽂件头，完整性检测
  5. ⾃带函数过滤
  6. WAF防护软件：宝塔、云盾等

ssrf

原理：利用网络请求的服务当跳板攻击内部其他服务

危害：

  1. 探测内⽹信息,⽤协议探ftp%26ip={ip}%26port={port}
  2. 攻击内⽹或本地其他服务
  3. 穿透防⽕墙

利用方式：

具体操作需要查看⽀持的协议，file协议查看⽂件、dict协议探测端⼝、ophergopher协议 ⽀持GET&POST请求，同时在攻击内⽹ftp、redis、telnet、Memcache上有极⼤作⽤利⽤ gopher协议访问redis反弹shell

漏洞存在的地方：

  1. 能够对外发起⽹络请求的地⽅
  2. 请求远程服务器资源的地⽅
  3. 数据库内置功能
  4. 邮件系统
  5. ⽂件处理
  6. 在线处理⼯具

举例：

  1. 在线识图，在线⽂档翻译，分享，订阅等，这些有的都会发起⽹络请求。
  2. 根据远程URL上传，静态资源图⽚等，这些会请求远程服务器的资源。
  3. 数据库的⽐如mongodb的copyDatabase函数
  4. 邮件系统就是接收邮件服务器地址这些地⽅
  5. ⽂件就找ImageMagick，xml这些
  6. 从URL关键字中寻找，⽐如：source,share,link,src,imageurl,target等

绕过姿势：

  1. http://example.com@127.0.0.1
  2. 利⽤IP地址的省略写法绕过,[::]绕过localhost
  3. DNS解析 http://127.0.0.1.xip.io/可以指向任意ip的域名：xip.io
  4. 利⽤⼋进制IP地址绕过,利⽤⼗六进制IP地址,绕过利⽤⼗进制的IP地址绕过

修复：

  1. 地址做⽩名单处理
  2. 域名识别IP 过滤内部IP
  3. 校验返回的内容对⽐是否与假定的⼀致

xss

原理：插入恶意脚本、实现对用户浏览器攻击

类型：存储、反射、dom

反射和dom区别：DOM-XSS是javascript处理输出、⽽反射性xss是后台程序处理

XSS绕过：

  1. ⼤⼩写
  2. js伪协议
  3. 没有分号
  4. Html5新标签
  5. 双层标签绕过

修复防御：

xxe

原理：解析用户传入的xml

作用：内⽹端⼝扫描、利⽤file协议等读取⽂件、攻击内⽹web应⽤使⽤get(struts2等)

危害：

  1. 加载恶意外部⽂件
  2. ⽂件读取
  3. 内⽹端⼝扫描
  4. 攻击内⽹⽹站
  5. 发起dos攻击等危害

防御：过滤⽤户提交的XML数据、libxml_disable_entity_loader设置为TRUE来禁⽤外部实体

如何快速判定xss类型

存储型xss：

发送⼀次带XSS代码的请求，以后这个⻚⾯的返回包⾥都会有XSS代码

反射型xss：

发送⼀次带XSS代码的请求，只能在当前返回的数据包中发现XSS代码

dom型xss：

发送⼀次带XSS代码的请求，在返回包⾥压根⼉就找不到XSS代码的影⼦

csrf、ssrf和重放攻击有什么区别

CSRF是跨站请求伪造攻击，由客户端发起

SSRF是服务器端请求伪造，由服务器发起

重放攻击是将截获的数据包进⾏重放，达到身份认证等⽬的

csrf和xss和xxe有什么区别，以及修复方式

XSS是跨站脚本攻击，⽤户提交的数据中可以构造代码来执⾏，从⽽实现窃取⽤户信息等攻击

修复⽅ 式：对字符实体进⾏转义、使⽤HTTP Only来禁⽌JavaScript读取Cookie值、输⼊时校验、浏览器与 Web应⽤端采⽤相同的字符编码

CSRF是跨站请求伪造攻击，XSS是实现CSRF的诸多⼿段中的⼀种，是由于没有在关键操作执⾏时进⾏ 是否由⽤户⾃愿发起的确认。

修复⽅式：筛选出需要防范CSRF的⻚⾯然后嵌⼊Token、再次输⼊密码、检验Referer

XXE是XML外部实体注⼊攻击，XML中可以通过调⽤实体来请求本地或者远程内容，和远程⽂件保护类 似，会引发相关安全问题，例如敏感⽂件读取。

修复⽅式：XML解析库在调⽤时严格禁⽌对外部实体的解析

shiro550与shiro721的区别，利⽤⽅式以及原理

  • 550可直接跑默认key，shiro721默认key跑不了，需要⼀个登录成功的cookie
  • Cookie中这个rememberMe字段，它是先对相关信息序列化，通过AES加密，再base64 编码，然后传⼊服务器，到服务器那⾥base64解码，AES解密，然后反序列化 AES这个Key曾经是固定的⼀个默认key，我们轻⽽易举能够构造恶意代码传⼊服务器，对于更 改过的key可以进⾏爆破爆出来，还是可以打这个反序列化
  • 使⽤ysoserial，通过检测出来的CC链⼦监听⼀个JRMP端⼝，我们向服务器传⼊的 payload就是指向这个端⼝，然后⼀旦Cookie发送到服务器，服务器连接到我们的JRMP服务 器，ysoserial⾃动通过CC链⼦把弹shell的命令打过去getshell

蚁剑（PHP用base64加密）

将蚁剑的正文内容进行URL解码后，流量最中明显的特征为@ini_set("display_errors","0");这段代码基本是所有WebShell客户端链接PHP类WebShell都有的一种代码，但是有的客户端会将这段编码或者加密，而蚁剑是明文，所以较好发现，同时蚁剑也有eval这种明显的特征。

由于蚁剑中包含了很多加密、绕过插件，所以导致很多流量被加密后无法识别，但是蚁剑混淆加密后还有一个比较明显的特征，即为参数名大多以“_0x.....=”这种形式（下划线可替换为其他）所以，以_0x开头的参数名，后面为加密数据的数据包也可识别为蚁剑的流量特征。

菜刀

最开始是明文传输，后来采用base64加密：

PHP类WebShell链接流量

如下：

第一：“eval”，eval函数用于执行传递的攻击payload，这是必不可少的；

第二：(base64_decode(POST[z0] ) ) ， ( base64decode(_POST[z0]))，(base64_decode( POST[z0]))，(base64 decode(_POST[z0]))将攻击payload进行Base64解码，因为菜刀默认是使用Base64编码，以避免被检测；

第三：&z0=QGluaV9zZXQ…，该部分是传递攻击payload，此参数z0对应$_POST[z0]接收到的数据，该参数值是使用Base64编码的，所以可以利用base64解码可以看到攻击明文。

注：

1.有少数时候eval方法会被assert方法替代。

2.post也会被_GET、$_REQUEST替代。

3.z0是菜刀默认的参数，这个地方也有可能被修改为其他参数名。

Windows中毒了 如何应急

检查系统账号安全

查看服务器是否有弱⼝令、可疑账号、隐藏账号、克隆账号、远 程管理端⼝是否对公⽹开放。

Win+R 打开运⾏，输⼊“eventvwr.msc”打开操作系统⽇志，查 看管理员登录时间、⽤户名是否存在异常

检查异常端⼝、进程

使⽤ netstat -ano 检查端⼝连接情况，是否有远程连接、可疑 连接（主要定位 ESTABLISHED）

根据 netstat 定位出的 pid，再通过 tasklist 命令进⾏进程定位 tasklist | findstr “PID”

也可以使⽤ D 盾_web 查杀⼯具、⽕绒剑、XueTr 等⼯具进⾏判断可疑进程（如蓝⾊、 红⾊进程、没有签名验证信息的进程、没有描述信息的进程、进程的属主、进程的路径是否合 法、 CPU 或内存资源占⽤⻓时间过⾼的进程）

检查启动项、计划任务、服务

检查服务器是否有异常的启动项，如：单击开始菜单 >【运⾏】，输⼊ msconfig看⼀下 启动项是否存在可疑启动，注册表run键值是否存在可疑启⽤⽂件，组策略，运⾏ gpedit.msc 查看脚本启动是否存在启⽤⽂件等

检查计划任务，如单击【开始】>【设置】>【控制⾯板】>【任务计划】，查看计划任务属 性，便可以发现⽊⻢⽂件的路径

检查服务⾃启动，如单击【开始】>【运⾏】，输⼊ services.msc，注意服务状态和启 动类型，检查是否有异常服务

检查系统相关信息

单击【开始】>【运⾏】，输⼊ systeminfo，查看系统信息是否打了补丁

查看⽤户⽬录，新建账号会在这个⽬录⽣成⼀个⽤户⽬录，查看 是否有新建⽤户⽬录

在服务器各个⽬录，可根据⽂件夹内⽂件列表时间进⾏排序，查 找可疑⽂件

⾃动化查杀

⽤360、卡巴斯基等病毒查杀系统病毒⽊⻢，Web 可以⽤ D 盾、河⻢ ⼯具查杀 Webshell 后⻔

⽇志分析

⽤ 360 星图⽇志分析⼯具进⾏分析攻击痕迹或⼿⼯结合 EmEditor 进 ⾏⽇志分析
